# Semantic Evaluation - Knowledge Graph Triple-to-Text Conversion

![Python Version](https://img.shields.io/badge/python-3.14+-blue.svg)

Semantic evaluation system that converts knowledge graph triples into fluent text using LLMs and evaluates quality using BLEURT.

## 📋 Description

This project implements a complete pipeline that:
1. Receives knowledge graph triples (subject, predicate, object) and a base text
2. Sends a prompt to Ollama's `/api/generate` endpoint
3. Compares the generated text with a reference using the BLEURT model

The code is organized following **Domain-Driven Design (DDD)** principles to facilitate maintenance, testing, and extension.

## 🏗️ Architecture

The project is structured in layers following DDD:

```
semantic_evaluation/
├── domain/                    # Domain layer
│   └── types.py              # Domain types (Triplet)
│
├── config/                    # Configuration
│   └── diplomat_config.py    # Service configuration
│
├── infrastructure/            # Infrastructure layer
│   ├── ollama_client.py      # Ollama API client
│   ├── bleurt_evaluator.py   # BLEURT evaluator
│   └── prompt_builder.py    # Prompt builder
│
├── application/               # Application layer
│   └── diplomat_service.py  # Main service (orchestration)
│
└── main.py                   # Entry point
```

## 🔧 Installation

### 1. Install Python Dependencies

```bash
pip install -r requirements.txt
```
### 2. Install and Configure Ollama

#### Install Ollama:
- **Windows/Mac**: Download from [https://ollama.ai](https://ollama.ai)

#### Download model:
```bash
ollama pull llama3:8b
```

#### Run model
```
ollama run llama3:8b
```

#### Verify Ollama is running:
```bash
ollama serve
```

The server should be available at `http://localhost:11434`

## 🚀 Usage

### Run Example

Simply run from the `semantic_evaluation` directory:

```bash
python main.py
```

### Use as a Library

```python
from application.diplomat_service import DiplomatService
from config.diplomat_config import DiplomatConfig

# Configure
config = DiplomatConfig(
    llama_model="llama3",
    ollama_url="http://localhost:11434/api/generate",
    bleurt_model_name="Elron/bleurt-large-512"
)

# Create service
diplomat = DiplomatService(config)

# Define triples
triples = [
    ("The university", "offers", "undergraduate programs"),
    ("The programs", "aim at", "critical student formation"),
    ("The institution", "implements", "continuous self-evaluation")
]

# Base text for style
base_text = (
    "The university provides broad academic programs guided by ongoing "
    "self-evaluation and a commitment to student development."
)

# Reference text for evaluation
reference = (
    "The university offers academic programs focused on critical student "
    "development and supported by continuous institutional self-assessment."
)

# Run complete pipeline
result = diplomat.run_pipeline(triples, base_text, reference)

print("Generated:", result["generated_text"])
print("Reference:", result["reference_text"])
print("BLEURT Score:", result["bleurt_score"])
```


## ⚙️ Configuration

The `DiplomatConfig` class allows customizing behavior:

```python
from config.diplomat_config import DiplomatConfig

config = DiplomatConfig(
    llama_model="llama3",                    # Ollama model to use
    ollama_url="http://localhost:11434/api/generate",  # Ollama URL
    bleurt_model_name="Elron/bleurt-large-512",  # BLEURT model
    device="cuda"                            # "cuda", "cpu" or None (auto)
)
```

**Parameters:**
- `llama_model` (str): Ollama model name (default: "llama3")
- `ollama_url` (str): Ollama endpoint URL (default: "http://localhost:11434/api/generate")
- `bleurt_model_name` (str): BLEURT model name on HuggingFace (default: "Elron/bleurt-large-512")
- `device` (str, optional): Device for BLEURT ("cuda", "cpu" or None for auto-detection)

## 📊 Understanding the Result

The `run_pipeline()` method returns a dictionary with:

```python
{
    "generated_text": str,    # Text generated by LLM
    "reference_text": str,    # Reference text provided
    "bleurt_score": float     # BLEURT score (higher is better)
}
```

**BLEURT Score:**
- Values typically between -1 and 1
- Higher values indicate greater semantic similarity
- Score > 0.5 generally indicates good quality

## 🔍 Detailed Structure

### Domain Layer
- **`types.py`**: Defines `Triplet = Tuple[str, str, str]` - domain type for triples

### Application Layer
- **`diplomat_service.py`**: Orchestrates the complete pipeline, coordinates infrastructure

### Infrastructure Layer
- **`ollama_client.py`**: HTTP client for communication with Ollama
- **`bleurt_evaluator.py`**: Loads and uses BLEURT model for evaluation
- **`prompt_builder.py`**: Builds formatted prompts for the LLM

### Config Layer
- **`diplomat_config.py`**: Centralizes all configurations

## 🐛 Troubleshooting

### Error: "Connection refused" when connecting to Ollama
- Verify Ollama is running: `ollama serve`
- Confirm the URL in `DiplomatConfig`

### Error: "Model not found" in Ollama
- Download the model: `ollama pull llama3:8b`
- Verify the model name in `DiplomatConfig`

### Error loading BLEURT
- First run downloads the model automatically (may take time)
- Verify internet connection
- For CPU, may be slow on first run

### CUDA out of memory
- Use `device="cpu"` in `DiplomatConfig`
- Or reduce BLEURT model size

### Import errors when running main.py
- Make sure you're running from the `semantic_evaluation` directory
- Ensure all dependencies are installed: `pip install -r requirements.txt`
